## 멀티프로세싱 vs 멀티코어 vs 멀티스레드

- **멀티프로세싱**: CPU 칩 자체가 여러 개 있는 것
- **멀티코어**: 하나의 CPU 칩 안에 여러 개의 코어가 있는 구조
- **멀티스레드**: 하나의 작업을 여러 개가 나눠서 동시에 처리할 수 있도록 하는 구조

처음에는 단일 CPU에서 멀티프로세싱으로 발전했고, 하나의 메인 보드에 여러 개의 CPU를 넣는 것은 비용 + 효율성 측면에서 부적합하여 CPU 하나에 여러 개의 코어를 넣기 시작하면서 멀티코어가 등장했다.

## 스레드

프로그램 안에서 여러 실행 흐름을 의미한다. 웹 서버가 단일 스레드 프로세스로 작동하면 한 번에 하나의 클라이언트만 서비스할 수 있다. 요청마다 새 프로세스를 생성하는 것은 오버헤드가 크므로, 한 프로세스 안에 여러 스레드를 만드는 멀티스레드 방식이 더 효율적이다.

대부분의 운영체제 커널도 다중 스레드이며, Linux 시스템에서 부트 동안 여러 커널 스레드가 생성되어 장치 관리, 메모리 관리, 인터럽트 처리 등의 작업을 수행한다.

## 병행성 vs 병렬성

- **병행성(Concurrency)**: 모든 작업이 진행되게 하여 둘 이상의 작업을 지원. 여러 작업이 동시에 실행되는 것처럼 보일 수 있으며, CPU 스케줄링이 중요함
- **병렬성(Parallelism)**: 둘 이상의 작업을 동시에 수행하는 것

병렬성 없이 병행성을 가질 수 있다.

## 병렬 실행의 유형

- **데이터 병렬 실행**: 동일한 데이터의 부분집합을 다수의 코어에 분배한 뒤 각 코어에서 동일한 연산을 실행
- **태스크 병렬 실행**: 서로 다른 작업을 여러 코어가 나누어 동시에 실행

## 스레드 매핑 모델

사용자 스레드는 커널이 인식하지 못하기 때문에 CPU 실행과 스케줄링을 위해 커널 스레드와 연결되어야 한다.

### 다대일 (Many-to-One)
- 많은 사용자 수준 스레드를 하나의 커널 스레드로 사상
- 스레드 관리는 사용자 공간의 스레드 라이브러리에 의해 수행되어 효율적 (커널 개입 없음, 문맥 교환 빠름)
- 단점: 한 스레드가 봉쇄형 시스템 콜을 하면 전체 프로세스가 봉쇄됨. 다중 코어에서 병렬 실행 불가

### 일대일 (One-to-One)
- 각 사용자 스레드를 각각 하나의 커널 스레드로 사상
- 장점: 하나의 스레드가 봉쇄되어도 다른 스레드 실행 가능, 더 많은 병렬성 제공
- 단점: 사용자 스레드가 많으면 커널 스레드도 많아져 커널 부담 증가

### 다대다 (Many-to-Many)
- 여러 사용자 수준 스레드를 그보다 작은 수 혹은 같은 수의 커널 스레드로 사상
- 일대일과 다대일의 중간 성질

## 스레드 풀 (Thread Pool)

미리 일정 개수의 스레드를 생성해 두고, 작업이 들어오면 그 스레드들이 일을 처리하도록 하는 방식이다. 스레드를 매번 생성하면 비용이 많이 들기 때문에 기존 스레드를 재사용한다.

- 스레드 풀 크기는 CPU 코어 수, 물리 메모리 용량, 동시 요청 클라이언트 최대 개수 등을 고려하여 결정
- 장점: 새 스레드 생성 비용 절감, 동시 존재 스레드의 최대 개수 제한 가능

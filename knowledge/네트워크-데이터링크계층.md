## 데이터 링크 계층 개요

- **데이터 링크 계층**: 패킷을 프레임으로 만들어 주는 계층 (MAC 주소 기반, 물리 주소)
- **네트워크 계층**: IP 주소 기반 (논리 주소)

## 이더넷 프레임

이더넷의 프레임화 규격은 크게 Ethernet2 규격과 IEEE802.3 규격 2종류이다.

Ethernet2는 데이터(패킷)에 '프리앰블', '헤더', '프레임 검사 순서'를 추가한다.
- **프리앰블**: 송신자와 수신자의 클럭을 맞추는 역할
- **MTU(Maximum Transmission Unit)**: 프레임으로 보낼 수 있는 데이터의 최대 크기는 1500바이트

## MAC 어드레스 통신 방식

### 유니캐스트
- 1:1 통신
- MAC 어드레스는 기본적으로 고유하기 때문에 노드에 대해서 1:1로 통신 가능

### 브로드캐스트
- 1:n 통신 (동일 네트워크 내의 모든 노드)
- 목적지 MAC 주소에 FF-FF-FF-FF-FF-FF를 넣음

### 멀티캐스트
- 1:n 통신 (특정 그룹에 포함된 노드)

## ARP (Address Resolution Protocol)

동일한 네트워크 내에서 노드 A가 노드 B와 통신하고 싶은데 노드 B의 MAC 주소를 모를 때:
1. 브로드캐스트로 모든 노드에게 해당 요청을 보냄
2. 요청을 받은 노드 B는 노드 A에게 자신의 MAC 주소를 알려줌
3. ARP 테이블에 매핑을 저장
4. 이후 유니캐스트로 통신

## MAC 어드레스 스위칭

L2 스위치는 메모리상에 'MAC 어드레스 테이블'(포트와 발신지 MAC 어드레스로 구성)을 유지한다.

- MAC 어드레스 테이블의 항목은 영구적이지 않음
- 포트에 연결된 케이블을 분리하거나 일정 시간 사용되지 않으면 삭제
- **에이징 타임**: 삭제까지의 시간 (시스코 카탈리스트 디폴트 300초/5분)
- 삭제 후 다시 요청이 들어오면 flooding(브로드캐스트)으로 처리

## VLAN과 Trunk Port

VLAN은 하나의 물리적 스위치를 논리적으로 분리하여 별도의 네트워크처럼 동작하게 한다. Trunk Port는 스위치 간에 여러 VLAN의 트래픽을 전달하기 위해 802.1Q 태그를 사용하여 같은 VLAN끼리 통신할 수 있게 한다.

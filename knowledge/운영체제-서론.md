OS는 여러 가지 프로그램을 프로세스라고 하는 단위로 실행한다. 리눅스는 디바이스 드라이버라고 하는 프로그램을 통해 디바이스를 다루며, 커널 모드로 동작할 때만 디바이스에 접근 가능하다. 디바이스 드라이버는 커널 모드로 동작하고, 프로세스는 사용자 모드로 동작한다.

운영체제에는 각 장치 컨트롤러마다 장치 드라이버가 있으며, CPU와 장치 컨트롤러는 병렬로 실행되어 메모리 사이클을 놓고 경쟁한다.

## 멀티프로세서 시스템

일반적인 다중 처리기 시스템(멀티프로세스)은 단일 코어 CPU가 있는 두 개 이상의 프로세서가 있다. 멀티프로세스를 사용하면 처리량이 늘어나지만 N개의 프로세서의 속도 향상 비율은 N이 아니다. 여러 프로세서가 하나의 작업을 협력할 때 모든 프로세서가 올바르게 작동하게 유지하는 데 일정한 양의 오버헤드가 발생하기 때문이다.

## SMP (Symmetric Multiprocessing)

대칭형 다중 처리 구조로, 메인 메모리를 공유하면서 각각 자체 CPU를 가지는 두 개의 프로세서가 있는 구조이다. N개의 CPU가 있으면 성능을 크게 저하하지 않으면서 N개의 프로세스를 실행할 수 있다. 단점은 각 CPU가 독립적이기 때문에 하나는 유휴 상태이고 다른 하나는 과부하가 걸려 비효율적일 수 있다.

## NUMA (Non-Uniform Memory Access)

CPU에 작고 빠른 로컬 버스를 통해 액세스되는 자체 로컬 메모리를 제공하는 아키텍처이다. 기존 UMA에서는 각각의 CPU가 하나의 메인 메모리를 공유했다면, NUMA는 각각의 CPU가 자신의 자체 로컬 메모리를 가진다. 모든 CPU가 공유 시스템 연결(interconnect)로 연결되어 있어 다른 CPU의 메모리에도 접근 가능하지만, interconnect를 통한 접근 시 지연 시간이 증가하여 성능 저하가 발생할 수 있다. CPU 스케줄링 및 메모리 관리를 통해 이 단점을 최소화할 수 있다.

## 시스템 콜

사용자 프로그램이 자신을 대신하여 운영체제가 수행하도록 지정되어 있는 작업을 운영체제에 요청할 수 있는 방법을 제공한다. 운영체제에 의해 사용 가능하게 된 서비스에 대한 인터페이스를 제공하며, 일반적으로 인터럽트 벡터의 특정 위치로 트랩을 거는 형태를 취한다. 운영체제가 컴퓨터의 제어를 얻을 때마다 항상 커널 모드(모드 비트 0)에 있다.

## 가상 메모리

다중 태스킹 시스템에서 운영체제는 적절한 응답 시간을 보장해야 하기 때문에 가장 좋은 방법은 가상 메모리이다. 여러 프로세스를 병행하게 실행하려면 각 프로세스에 대한 정보가 메모리에 저장되어 있기 때문에 메모리 관리와 CPU 스케줄링이 필요하다.

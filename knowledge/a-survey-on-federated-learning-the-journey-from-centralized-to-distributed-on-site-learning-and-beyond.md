- 원시 데이터를 장치에 보관하고 데이터 통신 오버헤드를 제거하면서 로컬 ML 학습을 포함한다
- 학습되고 공유된 모델의 연합은 중앙서버에서 수행되어 참가자 간에 구축된 지식을 집계하고 공유
- DL : deep learning
- 온사이트 ML 및 연합 학습이 중앙 집중식 시스템에 대한 대체 솔루션으로 발전
- 온디바이스 ML은 클라우드에서 장치로 ML작업을 푸시하여 원시 데이터를 로컬에 보관하지만 각 장치는 동료의 데이터와 경험으로부터 이익을 얻지 못하고 자체 모델을 구축 → 각 기기는 자기 데이터만 가지고 혼자 학습을 한다( 장치끼리 서로 배운것을 공유하지 못한다 )
- FL은 개인 정보를 보호하고 막대한 데이터 수집 오버헤드를 줄이면서 이러한 문제를 극복하기 위해 도입됨 ( 개인 정보 보호가 핵심임 )
- FL의 목적은 개인정보를 서버에 보내지 않으면서 중앙집중식에 가까운 성능을 얻는것
- 중앙 서버에서는 분산 학습의 이점을 누릴 수 있게 로컬에서 계산된 업데이트 및 분석 결과만 수신 및 집계되어 향상된 글로벌 모델을 구축한다
- 그런다음 새로운 모델이 클라이언트와 공유되어 지식을 공유한다
- Iaas , Saas , Paas
- 클라우드/중앙 집중식 기반 접근 방식에는 다음과 같은 다른 문제도 발생합니다. 1) 데이터가 클라우드에 도달하기 위해 수백, 심지어 수천 마일 떨어진 곳으로 전송될 수 있으므로 지연 시간이 발생하고 2) 데이터를 네트워크를 통해 클라우드 컴퓨팅으로 이동하는 데 비용이 무료가 아니므로 데이터 전송 비용이 발생합니다. 이러한 문제를 극복하기 위해 일부 ML 작업을 강력한 리소스를 갖춘 장치로 이동하는 온사이트 ML이 발전했습니다.
- 온사이트 ML : 자체시설(온사이트) 안에서 클라우드로 데이터를 보내지 않고 로컬 서버-사내 인프라에서 직접 ML 학습/추론을 수행하는 방식
- FL은 원시 데이터들을 온디바이스에 보관하고 로컬에서 계산된 모델 업데이트를 집계하여 데이터 개인 정보를 보호하고 데이터 통신 오버헤드를 줄인다
- FL은 데이터를 서버로 절대 보내지 않고 , 기기 내부에서 학습한 “모델 업데이트” 만 서버로 보낸다
- grediant(기울기) : 오차(Loss) / 가중치 (Weight)
- 경사하강법은 가중치가 바뀌면 Loss 가 어떻게 변하는가를 이용해서 가중치를 조정하는 알고리즘
